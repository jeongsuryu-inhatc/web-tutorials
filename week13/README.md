# 13주차 — 생성형 AI와 바이브 코딩

핵심: **개발의 주도권은 개발자에게 있다.** AI는 뛰어난 도구이지, 결정권자가 아니다.
AI에게 '시키고', 결과를 '검증하고', 최종 책임을 '내가 지는' 흐름을 연습한다.

## 실습 흐름 (11주차 Todo 앱에 기능 추가)

## 좋은 지시(프롬프트)의 예 — 개발자가 주도권을 갖는 방식

나쁜 예 (주도권을 AI에게 넘김):
> "todo 앱 좋게 만들어줘"

좋은 예 (개발자가 결정하고 AI에게 시킴):
> "Express + EJS + MySQL Todo 앱에 검색 기능을 추가해줘.
> 조건: (1) GET /todos?q=키워드 방식 (2) SQL은 LIKE와 플레이스홀더(?) 사용 — 문자열 연결 금지
> (3) 기존 filter 쿼리와 함께 동작할 것 (4) routes/todos.js만 수정하고 수정된 부분만 보여줘"

## 검증 체크리스트 (AI 코드를 믿지 말 것)
- [ ] SQL 인젝션: 쿼리에 사용자 입력을 문자열로 붙이지 않았는가? (플레이스홀더 `?` 사용)
- [ ] 존재하지 않는 함수/컬럼을 쓰지 않았는가?
- [ ] 빈 입력, 특수문자 입력 시 동작하는가?
- [ ] 기존 기능(필터, 수정, 삭제)이 여전히 동작하는가?
- [ ] 내가 이 코드를 설명할 수 있는가? ← 가장 중요
