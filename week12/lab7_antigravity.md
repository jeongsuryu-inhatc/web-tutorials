# 실습 7 — Google Antigravity 설치와 특징 실습

Antigravity는 챗봇이 아니라 **에이전트 기반 개발 플랫폼**이다. Google 계정만 있으면 무료로 Gemini 3 Pro 에이전트를 쓸 수 있다.

## 7-1. 설치

1. https://antigravity.google 접속 → 본인 OS(Windows/Mac/Linux)용 설치 파일 다운로드
2. 설치 후 Google 계정으로 로그인
3. 첫 실행 시 안내에 따라 **Chrome 확장 프로그램**을 설치한다 (에이전트가 브라우저로 결과를 직접 테스트하는 데 필요)
4. 에디터 테마·단축키 설정은 VS Code와 거의 같다 (VS Code 계열)

## 7-2. 특징 실습 ① — Agent Manager에게 작업 시키기

1. `week12` 폴더를 Antigravity로 연다.
2. Agent Manager에 작업을 지시한다.
   ```
   ai_history.html 파일을 새로 만들어줘.
   AI 역사(다트머스 회의, 딥러닝, 알파고, 트랜스포머, 생성형 AI)를 카드 형태로 보여주는
   단일 HTML 파일. 외부 라이브러리 금지, 반응형, 파란색 포인트의 미니멀 디자인.
   ```
3. 에이전트가 **계획 → 파일 생성 → 검증**하는 과정을 관찰하고, 사람이 개입(승인/수정 지시)할 수 있는 지점을 확인한다.

## 7-3. 특징 실습 ② — 브라우저 통합 테스트

1. 작업 완료 후 에이전트에게 요청한다.
   ```
   방금 만든 페이지를 브라우저로 열어서 모바일 크기에서도 잘 보이는지 확인하고,
   문제가 있으면 고쳐줘.
   ```
2. 에이전트가 Chrome을 직접 조작해 확인하는 과정을 관찰한다. (챗봇과의 가장 큰 차이!)

## 7-4. 특징 실습 ③ — Fast vs Planning 모드 비교

- **Fast 모드**로: "페이지 제목의 색을 초록색으로 바꿔줘" (간단 수정)
- **Planning 모드**로: "카드에 연도별 필터 기능을 추가해줘" (설계가 필요한 작업)
- 두 모드의 진행 방식 차이를 한 줄로 기록한다.

## 7-5. 제출물

- 생성된 `ai_history.html`
- 에이전트가 브라우저를 조작하는 장면 스크린샷
- Fast/Planning 모드 차이 메모
